# MVP 路线图

更新时间：`2026-04-21`

本文件保留产品视角下的 MVP 分层，用来回答“首版先做到哪一层”。
详细实施阶段、依赖关系和子阶段拆分以 [06-phased-implementation-plan.md](./06-phased-implementation-plan.md) 为准。

## 产品层级

### 层级 0：方向确认

目标是定清楚“首版必须有、可以延后、不做”的边界。

对应已完成工作：

- 技术选型
- 目录结构
- 生成 DSL 草案
- 通用模块范围

### 层级 1：平台底座

目标是让平台具备可运行、可扩展、可上线的基础工程能力。

当前映射到实施计划：

- `Phase 1` 垂直切片闭环
- `Phase 2` 认证与权限底座
- `Phase 3` 中的后台布局与核心管理模块

用户能感知到的能力：

- 服务端和前端骨架可运行
- 能登录
- 能根据权限展示菜单和页面
- 有统一的接口、权限和审计基础
- Vue 端同时具备企业预设和自建风格预设两条可演进路线

### 层级 2：标准模块生成

目标是让新增一个标准业务模块不再从零手写 CRUD 骨架。

当前映射到实施计划：

- `Phase 4` 代码生成成熟化

用户能感知到的能力：

- 用 entity schema 生成 server、页面和权限点
- 生成代码可直接运行并进入二次维护
- 升级生成时不会直接覆盖手写修改

### 层级 3：AI 辅助

目标是让 AI 参与规格生成和局部建议，但不取代工程约束。

当前映射到实施计划：

- `Phase 5` AI 辅助开发

用户能感知到的能力：

- 自然语言转 schema
- 字段、查询、表单和测试建议
- AI 输出可校验、可回放、可人工接管

### 层级 4：生产强化与平台扩展

目标是把平台从“可用”推进到“可规模化使用和扩展”。

当前映射到实施计划：

- `Phase 6` 生产强化
- `Phase 7` 平台扩展

用户能感知到的能力：

- Docker 部署、观测、安全、E2E
- 多租户、数据权限、定时任务等企业增强能力
- React 适配层、Studio、插件机制和模板市场
- 企业预设与自建预设逐步沉淀为稳定主题与组件体系

## 首版明确不做

- 可视化拖拽低代码
- 复杂工作流引擎
- 多前端框架运行时无损切换
- 全自动 AI 改造旧项目
- “一句话生成完整 ERP”
