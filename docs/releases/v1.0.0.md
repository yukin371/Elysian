# v1.0.0 Release Notes

发布日期：`2026-05-14`

发布对象：

- tag: `v1.0.0`
- release commit: `b0c2c0f`
- post-release archive head: `d03e3ff`

## 本次发布包含

- 首个参考发行版闭环：`apps/example-vue`、`apps/server`、`packages/persistence`、`packages/generator` 已收口为可复制、可验证、可上线的新项目 starter 基线
- 仓库发布门槛闭环：`bun run check`、`bun run build:vue`、`bun run server:image:verify`、`bun run e2e:smoke:full`、`bun run e2e:tenant:full` 已固定为首发验证命令链
- go-live 门禁闭环：`go-live:report`、`go-live:handoff`、`go-live:gate`、`go-live:finalize` 已形成统一执行口径
- 多租户安全修复：server runtime 改为优先使用 `DATABASE_RUNTIME_URL`，允许运行态数据库连接切到非 owner、`NOBYPASSRLS` 的受限角色，确保 PostgreSQL RLS 对跨租户隔离真正生效
- staging 演练闭环：`health`、`metrics`、管理员登录、权限 gate、核心列表/写操作、super-admin 租户访问、tenant admin 禁止访问、非默认 tenant 登录、跨租户隔离共 `10` 项验证全部通过

## 关键变更

### 1. 运行态数据库角色分离

- `DATABASE_URL` 继续保留给 migration / admin 路径
- `DATABASE_RUNTIME_URL` 新增为 server runtime 首选连接串
- 运行态不再默认复用 table owner 身份访问业务表

### 2. 发布与演练材料收口

- 已补齐候选包、验收包、go-live 准备包、分角色 handoff 与 M1-M4 执行清单
- 发布对象、tag、环境输入与最终 gate 结论已归档到 `docs/plans/*` 与 `artifacts/go-live/*`

### 3. 参考发行版交付基线

- 后台常用模块、generator session、标准 CRUD 前端 surface 与最小生产烟测已形成首发交付基线
- 当前仓库不再停留在“首发前准备”，而是已完成 `v1.0.0` 发布对象锁定
