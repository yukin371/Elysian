## Summary

- 变更目标：完成首个参考发行版 `v1.0.0` 的发布闭环，将 `apps/example-vue`、`apps/server`、`packages/persistence`、`packages/generator` 收口成可复制、可验证、可上线的新项目 starter 基线。
- 关键改动：
  - 收口参考发行版候选包、验收包、go-live 准备包、M1-M4 执行清单与分角色 handoff。
  - 新增 runtime DB 身份分离，server runtime 优先使用 `DATABASE_RUNTIME_URL`，migration / admin 继续使用 `DATABASE_URL`。
  - 修复多租户跨租户隔离阻断，确保 PostgreSQL RLS 对非 owner、`NOBYPASSRLS` 运行角色真正生效。
  - 完成 `staging` go-live 演练，`health`、`metrics`、管理员登录、权限 gate、核心列表/写操作、super-admin 租户访问、tenant admin 禁止访问、非默认 tenant 登录、跨租户隔离共 `10` 项检查全部通过。
  - 锁定正式发布对象：`v1.0.0 -> b0c2c0f`，并补齐 release notes。
- 边界说明：
  - `b0c2c0f` 是正式发布对象。

## Risk

- 风险点：
  - 当前 `staging` 演练已无 blocker，但如果后续切换到新的真实目标环境，仍需按同一模板重新锁定 runtime DB role、backup / restore 和 `M3` 冒烟证据。
  - 远端推送当前受本机网络阻断，`git push origin dev` / `git push origin v1.0.0` 需在网络恢复后执行。
- 回滚方式：
  - 仓库发布层：回退到 `v1.0.0` tag 之前的 `main` 基线。
  - 环境层：按 `docs/plans/2026-05-06-go-live-preparation-packet.md` 中的 backup / restore / rollback 边界执行。
