# 2026-04-22 Phase 4D Safe Apply Kickoff

## 背景

- `Phase 4` 目前仅完成 `P4A` 预验证，尚未完成 apply/merge 安全收口。
- 现有 generator 冲突策略为 `skip/overwrite/fail`，缺少“安全覆盖受管生成文件”的中间策略。

## 目标

先完成 `P4D` 的最小安全基线：二次生成时默认不误覆盖手写文件，同时保留可控覆盖路径。

## 本轮范围

- 在 generator 写入流程新增 `overwrite-generated-only` 冲突策略。
- 仅当目标文件包含生成标记（`@generated by elysian-generator`）时允许覆盖。
- CLI 与测试同步支持新策略。
- 将“是否冲突”从 `size > 0` 调整为“文件是否存在”，把空文件纳入冲突策略。
- 冲突策略执行改为“先预检全部冲突，再执行写入”，避免中途失败导致部分写入。
- 文件与 manifest 采用 temp + rename 原子替换，降低写入中断导致的损坏风险。

## 验证

- `packages/generator/src/write.test.ts` 覆盖：
  - 受管文件可覆盖
  - 非受管文件拒绝覆盖
  - 空文件在 `skip/fail/overwrite-generated-only` 下行为一致
  - 冲突发生时无部分写入
  - 写入后无 `.tmp-*` 临时文件残留
- `packages/generator/src/cli-args.test.ts` 覆盖 `--conflict overwrite-generated-only` 及参数优先级行为
- `scripts/e2e-generator-safe-apply.ts` 覆盖受管覆盖成功 / 非受管拒绝 / skip 保守写入三场景
- 全量 `bun run check` 通过。

## 收口状态

- 结论：`P4D` 已完成，可进入 `P4E` 回归验证阶段。
- 仍保留边界：暂不进入三向语义合并与 section-level merge。

## 不在范围

- 三向语义合并
- 细粒度区域保护（section-level merge）
- 可视化 diff / apply UI
