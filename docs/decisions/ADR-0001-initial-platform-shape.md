# ADR-0001 初始平台形态

状态：`accepted`

日期：`2026-04-20`

## 背景

仓库当前仍是绿地阶段，但项目目标已经明确为“企业级快速开发平台”，而不是单一前端脚手架或纯原型生成器。需要尽早固定一批长期边界，避免后续实现漂移。

## 决策

1. 后端内核以 `Elysia` 为中心。
2. 前端不绑定单一框架，按适配层支持 `Vue` 与 `React`。
3. 代码生成以 schema 驱动为主，AI 作为结构化规格生成和实现辅助，而不是直接生成黑盒项目。
4. 首版优先企业后台与中后台场景，不把 SSR 作为首发硬约束。
5. 平台优先生成可维护、可测试、可部署的正常源码。

## 影响

- 仓库结构会偏向 `server + schema + generator + frontend adapters`
- 技术选型需要优先服务于契约统一和再生成能力
- 前端相关决策必须避免把平台核心耦合到单一框架

## 代价

- 首版抽象成本更高
- 双前端适配意味着模板和测试成本上升
- 需要更严格的 owner 和边界管理

## 暂不决策项

- ORM 与数据库策略
- 工作区工具
- 首版前端先落单实现还是双示例
- 部署与 CI 方案

这些保留到后续 ADR。
