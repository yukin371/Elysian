# DESIGN

本文件定义 `apps/example-uniapp` 的界面约束。

它不替代 [packages/ui-enterprise-vue/DESIGN.md](/E:/Github/Elysian/packages/ui-enterprise-vue/DESIGN.md)，而是把 `uniapp` 第二 C 端在当前阶段允许使用的展示规则收紧到与主线一致的产品感。

## 1. 文档关系

- `packages/ui-enterprise-vue/DESIGN.md` 仍是仓库共享视觉基线。
- 本文件只补 `apps/example-uniapp` 的移动端页面壳、表单、通知卡片和个人中心展示规则。
- 若两者冲突，以共享设计文档为准。

## 2. 当前定位

- `apps/example-uniapp` 是第二优先级 C 端示例，不是独立品牌站点。
- 当前页面目标是验证登录、会话恢复、通知列表与通知详情闭环。
- 即使处于验证阶段，默认可见文案也必须保持产品化表达，不暴露“骨架”“占位”“后续再接”这类研发态措辞。
- `apps/example-uniapp` 可以和 B 端应用不同，但 C 端应用内部必须保持单一移动端风格，不允许页面各自漂移。

## 3. 视觉原则

- 保持 `背景层 -> 主内容层 -> 局部分组` 三层以内，不堆叠多层卡片。
- 延续企业预设的单主轴蓝色体系，不在 `uniapp` 侧额外引入第二套青绿色主色。
- 优先靠边框、浅底和留白建立层级，弱化重阴影。
- 同一应用内，登录、个人中心、通知列表、通知详情必须共享同一套主题色、圆角、按钮和面板语义。
- 圆角统一收敛到：
  - 输入框、按钮：`24rpx`
  - 主面板、通知卡片：`32rpx`
  - 状态胶囊：`999rpx`

## 4. 色彩规则

- 主强调色：`#2457d6`
- 主强调色 hover / pressed：`#1d4ed8` / `#173ea6`
- 正文主色：`#0f172a`
- 次级文字：`#475569`
- 弱化文字：`#64748b`
- 面板边框：`rgba(15, 23, 42, 0.08)`
- 页面背景：以浅灰蓝底为主，可加低对比渐变，但不做高饱和装饰
- 上述颜色视为本应用主题 token；不允许某个页面单独切成新的主色、成功色主轴或背景体系

## 5. 组件规则

- `ShellPanel` 作为 app 内通用主面板，默认使用浅边框、轻阴影和统一圆角，不再自行派生第二套容器视觉。
- 登录页、个人中心、通知列表、通知详情应复用同一套按钮、输入和状态胶囊语义。
- 通知列表卡片允许使用轻量信息块，但不应比主面板更抢眼。
- 若后续新增页面，默认继续沿用现有面板、按钮、输入和状态语义；除非先更新本文件，否则不新增第二套样式语言。

## 6. 文案规则

- 默认语言：`zh-CN`。
- 标题和说明文案应描述用户当前能做什么，不描述实现过程。
- 全应用文案语气保持一致：简洁、产品化、面向用户动作，不混入后台术语和研发态说明。
- 不使用：
  - `骨架`
  - `占位`
  - `POC`
  - `demo only`
  - `后续接入`

## 7. 实施优先级

1. 先统一 `uni.scss` token
2. 再统一 `ShellPanel`、按钮、输入框的圆角/边框/色彩
3. 再清理研发态文案
4. 最后才调整局部装饰和背景氛围
