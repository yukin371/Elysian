# DESIGN

本文件是 `apps/example-vue` 的补充设计文档。

它不替代 [packages/ui-enterprise-vue/DESIGN.md](/E:/Github/Elysian/packages/ui-enterprise-vue/DESIGN.md)，只定义示例应用在展示层允许做的补充与限制。

若任务需要按用户使用路径整理具体页面，还应继续阅读 [apps/example-vue/PAGE_DESIGN.md](/E:/Github/Elysian/apps/example-vue/PAGE_DESIGN.md)。该文档只负责页面设计优化，不覆盖本文件。

## 1. 文档关系

- `packages/ui-enterprise-vue/DESIGN.md` 是共享企业预设的主规则。
- 本文件只约束 `apps/example-vue` 的首页、对照区块、说明性面板和演示包装层。
- `apps/example-vue/PAGE_DESIGN.md` 用用户任务整理页面结构优化。
- 若与共享预设规则冲突，以 `packages/ui-enterprise-vue/DESIGN.md` 为准。

## 2. 示例应用定位

- 这是示例应用，不是独立品牌站点。
- 目标是展示协议层、企业预设和真实工作区闭环，而不是做营销型 landing page。
- 允许有少量“导览式”内容，但不能压过实际工作区。
- 本应用内部必须保持单一后台风格，不因为页面类型不同而切换第二套主题语言。

## 3. 允许的展示差异

- 可保留更深色的页面外层背景，用来区分“示例壳”与内部企业工作区。
- 可保留深色玻璃感的外层面板，但主面板仍应收敛到共享尺度附近：
  - 外层主面板圆角不超过 `16px`
  - 外层主面板阴影保持轻量，不使用重压式大阴影
- 可保留少量导览面板，例如：
  - 工作区地图
  - Schema 契约说明
  - 权限快照
- 可在首页上展示“共享协议”“示例说明”“当前状态”等辅助信息。
- 这些差异只允许作为应用级一致差异存在；一旦采用，相关页面应统一沿用，而不是只在个别页面出现。

## 4. 禁止事项

- 不允许为示例页重新定义一整套与企业预设冲突的颜色、圆角、按钮语义。
- 不允许通过大量卡片堆叠制造“内容丰富感”。
- 不允许把 `POC`、`demo only`、`preview skeleton` 作为默认主界面标签长期展示。
- 不允许让说明区比真实工作区更吸引注意力。
- 不允许同一个应用里一部分页面走企业后台蓝色体系，另一部分页面切成新的品牌色或新的圆角尺度。
- 不允许把外层展示壳继续推向更大的圆角或更重的阴影，以至于和内部企业工作区形成两套割裂的产品感。

## 5. 布局优先级

- 第一视觉焦点应是“真实可运行工作区”。
- 左侧导览区是辅助，不应在面积、对比度或视觉噪声上压过右侧工作区。
- 若信息过多，优先删减说明面板，而不是继续增加容器层。
- 首屏不做宣传式 hero；默认直接进入后台壳层、导航和当前功能工作区。
- 未登录时应使用独立访问会话页，不渲染后台侧边栏、工作区页签或伪菜单，避免把受保护导航误解为未接通功能。
- 运行状态、语言切换、环境信息只能作为紧凑顶条或侧栏辅助信息，不应抢占功能区。
- 运行态只能作为紧凑顶条或诊断辅助信息，不应作为与 `工作区` 并列的平级页签。
- 不允许使用“标题 + 副标题/说明段落”去解释工作区功能；首屏语义应改用结构化事实信息，例如标签、计数、状态、版本、权限或空状态，而不是一段解释文本。
- 标题旁的解释性标签、状态摘要和筛选说明默认视为冗余，除非它们直接影响操作，否则应删掉而不是堆在首屏。
- 说明文案只允许做最后兜底，不允许用来替代布局；若某页面的功能语义无法通过标题、列表、筛选、元信息或空状态直接看懂，应优先调整结构，不要用更长的说明文案去补救。
- generator 和 workflow 这类页面默认按“输入 / 概览 / 列表 / 详情”组织，不用标题和副标题去解释功能。
- 当页面同时存在“人类边界”和“AI 编辑”两种输入方式时，必须把入口并列摆在首屏，不允许只靠一段文案提示用户。
- 页面首屏可以保留少量结构化数据块，例如计数、状态分布、步骤条或轻量条形图，但不能把图表做成装饰。
- 加载提示应尽量短，优先让用户看到当前状态、可操作入口和结果列表，而不是先看到长句说明。
- 如果某页面的功能意图无法从布局、列表和状态块中直接看出来，优先重排结构，不要通过继续增加说明文字来补救。
- 页面可以参考若依这类后台的信息密度、导航优先级与操作路径，但不复刻其视觉细节或技术结构。
- 未登录时不展示伪造的系统菜单或 mock 模块入口；导航应来自真实 identity/menu 契约，登录入口只承担会话进入职责。

## 6. 文案规则

- 默认语言仍为 `zh-CN`。
- 示例页可以解释实现边界，但应使用产品化表达，不使用内部开发黑话。
- 英文回退文案要和中文语义一致，不保留过期临时术语。

## 7. 实施建议

当调整 `apps/example-vue` 展示层时，按以下顺序处理：

1. 先确认共享视觉 token 没有被覆盖
2. 再确认示例页是否真的需要额外导览面板
3. 如果只是分组说明，优先减容器，不新增完整卡片
4. 如果视觉冲突来自共享预设，应回到 `packages/ui-enterprise-vue` 修正，而不是只在示例页打补丁
