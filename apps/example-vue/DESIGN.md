# DESIGN

本文件是 `apps/example-vue` 的补充设计文档。

它不替代 [packages/ui-enterprise-vue/DESIGN.md](/E:/Github/Elysian/packages/ui-enterprise-vue/DESIGN.md)，只定义示例应用在展示层允许做的补充与限制。

## 1. 文档关系

- `packages/ui-enterprise-vue/DESIGN.md` 是共享企业预设的主规则。
- 本文件只约束 `apps/example-vue` 的首页、对照区块、说明性面板和演示包装层。
- 若与共享预设规则冲突，以 `packages/ui-enterprise-vue/DESIGN.md` 为准。

## 2. 示例应用定位

- 这是示例应用，不是独立品牌站点。
- 目标是展示协议层、企业预设和真实工作区闭环，而不是做营销型 landing page。
- 允许有少量“导览式”内容，但不能压过实际工作区。

## 3. 允许的展示差异

- 可保留更深色的页面外层背景，用来区分“示例壳”与内部企业工作区。
- 可保留少量导览面板，例如：
  - 工作区地图
  - Schema 契约说明
  - 权限快照
- 可在首页上展示“共享协议”“示例说明”“当前状态”等辅助信息。

## 4. 禁止事项

- 不允许为示例页重新定义一整套与企业预设冲突的颜色、圆角、按钮语义。
- 不允许通过大量卡片堆叠制造“内容丰富感”。
- 不允许把 `POC`、`demo only`、`preview skeleton` 作为默认主界面标签长期展示。
- 不允许让说明区比真实工作区更吸引注意力。

## 5. 布局优先级

- 第一视觉焦点应是“真实可运行工作区”。
- 左侧导览区是辅助，不应在面积、对比度或视觉噪声上压过右侧工作区。
- 若信息过多，优先删减说明面板，而不是继续增加容器层。

## 6. 文案规则

- 默认语言仍为 `zh-CN`。
- 示例页可以解释实现边界，但应使用产品化表达，不使用内部开发黑话。
- 英文回退文案要和中文语义一致，不保留过期临时术语。

## 7. 实施建议

当调整 `apps/example-vue` 展示层时，按以下顺序处理：

1. 先确认共享视觉 token 没有被覆盖
2. 再确认示例页是否真的需要额外导览面板
3. 如果只是分组说明，优先减容器，不新增完整卡片
4. 如果视觉冲突来自共享预设，应回到 `packages/ui-enterprise-vue` 修正，而不是只在示例页打补丁
