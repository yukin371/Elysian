# DESIGN

本文件定义 `@elysian/ui-enterprise-vue` 当前前端界面的默认设计约束。目标不是做一套抽象设计系统文档，而是给协作者与 agent 一份可直接执行的界面规则。

若任务同时涉及 `apps/example-vue` 的展示包装层，还应继续阅读 [apps/example-vue/DESIGN.md](/E:/Github/Elysian/apps/example-vue/DESIGN.md)。该文档只补充示例应用差异，不覆盖本文件。

## 1. 产品语境

- 默认用户群体以国内企业后台用户为主。
- 默认语言：`zh-CN`。
- 英文文案只作为回退，不应让英文成为默认可见主文案。
- 界面定位：克制、稳定、清晰，不做夸张的营销页视觉。

## 2. 视觉原则

- 一个页面最多保持 `背景层 -> 主面层 -> 局部功能层` 三层结构。
- 避免卡片套卡片；内层若只是分组，优先用留白、分隔线、浅底色，而不是再加完整卡片边框和阴影。
- 圆角使用统一尺度，不允许大面积 `22px+` 与 `999px` 混用。
- 强调色只保留一个主轴，避免蓝、紫、青、绿同时争抢注意力。
- 页面首先服务信息密度和可读性，而不是展示组件效果。
- 同一应用内的页面必须共享同一套主题色、圆角、阴影和容器语义，不允许首页一种风格、详情页另一种风格。
- 不同应用可以有不同风格，但差异必须先落在对应 `DESIGN.md`，不能由单个页面自行决定。

## 3. 色彩规则

- 主强调色：`#2457d6`
- 主强调色 hover / pressed：`#1d4ed8` / `#173ea6`
- 正文主色：`#0f172a`
- 次级文字：`#475569`
- 弱化文字：`#64748b`
- 分割线 / 边框：`rgba(15, 23, 42, 0.08)` 到 `rgba(15, 23, 42, 0.12)`
- 主内容面：`#ffffff` 或接近白色的低对比浅底
- 页面背景允许深色，但只能作为外层氛围，不能与内部功能区形成两套完全不同的产品感
- 以上颜色视为当前应用默认主题 token；若某应用要偏离，必须在自己的 `DESIGN.md` 中整体定义，而不是单页改色

禁用项：

- 默认紫色渐变作为品牌色
- 同页出现多个高饱和强调色并列争抢
- 依赖大面积发光阴影制造层级

## 4. 圆角与阴影

- 小控件：`8px`
- 输入框、按钮、局部容器：`12px`
- 主面板、工作区：`16px`
- 只有头像、小型状态胶囊允许 `999px`
- 默认阴影应弱化，优先使用边框和底色差异建立层级
- 以上圆角与阴影视为当前应用默认 token；同一应用内不允许按页面单独扩出第二套半径体系

禁用项：

- 为几乎所有卡片都加大圆角和重阴影
- 在同一块工作区内同时出现 `12 / 18 / 22 / 28 / 999` 的混乱半径体系

## 5. 容器规则

- 查询区、表格区、侧栏信息区可以是独立 surface，但要属于同一套视觉系统。
- 如果外层已经是完整工作区容器，内层信息块默认使用浅底分组，不再重复完整卡片语义。
- 统计卡片数量应克制；若只是展示单个数字，优先做轻量信息块。
- 空状态、告警、危险操作区可以提升语义强度，但仍使用同一色板。
- 同一应用内的列表页、详情页、表单页应复用一致的容器语义，不能这一页靠重卡片、下一页靠纯浅底、再下一页靠大阴影。

## 6. 文案与 i18n

- 默认中文文案应完整可读，不夹带内部工程术语。
- `POC`、`preview skeleton`、`demo only` 这类研发态词汇不应作为默认主界面标签长期存在。
- 中英文文案语义保持一致，避免中文是正式文案、英文还是临时占位。
- 同一应用内的页面文案语气应保持一致，不允许同一应用里同时出现产品化表达与研发态表达混用。

## 7. Owner 约束

- `packages/ui-enterprise-vue` 负责共享企业壳层与通用预设组件的视觉基线。
- `apps/example-vue` 只负责示例页自身的展示补充，不应重新定义一套相互冲突的全局视觉语言。
- 不把第三方组件库的视觉细节反向泄漏到 `ui-core`。

## 8. 实施优先级

当现有页面需要调整时，按以下顺序处理：

1. 先统一颜色、边框、圆角 token
2. 再减少冗余容器层级
3. 再处理局部装饰、徽标、渐变和阴影
4. 最后再补充少量必要的强调样式
